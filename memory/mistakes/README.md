# mistakes/ - 错误记录目录

记录AI犯错情况，提炼改进规则，防止重复犯错。

---

## 目录结构

```
mistakes/
├── YYYY-MM-DD-<topic>.md    # 错误记录文件
└── reviews/                 # 周回顾
    └── YYYY-W<周数>.md
```

---

## 如何使用

### 记录错误

当AI犯错时，使用mistake-tracking skill：

```
刚才AI建议使用Java 11+ API，但本项目用Java 8，请记录这个错误
```

### 查看错误

查看历史错误记录：

```
查看memory/mistakes/目录下的错误记录
```

### 定期回顾

每周回顾错误并提炼规则：

```
生成本周的错误的统计报告
```

---

## 文件命名规则

**错误记录**：`YYYY-MM-DD-<简短描述>.md`
- 示例：`2026-03-05-java-version.md`

**周回顾**：`YYYY-W<周数>.md`
- 示例：`2026-W10.md`（2026年第10周）

---

## 相关Skill

- **mistake-tracking** - 记录和分析错误
- **code-review** - 发现代码问题

## 最佳实践

1. 及时记录 - 发现错误立即记录
2. 具体描述 - 记录场景和正确做法
3. 提炼规则 - 从错误中学习
4. 定期回顾 - 每周分析模式
5. 持续更新 - 保持规则最新
