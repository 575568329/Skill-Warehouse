# Code Review - 代码审查

系统化的代码审查流程，确保代码质量和安全性。

## 审查流程

### 1. 收集代码变更

**方式A：审查当前变更**
```bash
# 查看未提交的变更
git diff

# 查看已暂存的变更
git diff --cached
```

**方式B：审查分支差异**
```bash
# 对比当前分支与master
git diff master...HEAD

# 对比两个特定commit
git diff <commit1> <commit2>
```

**方式C：审查特定文件**
```bash
# 只审查某些文件
git diff -- path/to/file.py
```

### 2. 多维度检查

按照以下维度逐一检查代码：

#### 🔴 Critical（必须修复）

**安全漏洞**
- SQL注入风险
- XSS跨站脚本攻击
- 硬编码密钥/密码
- 不安全的反序列化
- 权限校验缺失

**数据安全**
- 空值未防御导致白屏/崩溃
- 敏感信息泄露（日志、错误消息）
- 文件路径未验证

**核心逻辑**
- 业务逻辑错误
- 数据流断裂

#### 🟡 Important（建议修复）

**性能问题**
- N+1查询
- 内存泄漏风险
- 不必要的循环/重复计算
- 大对象未释放

**代码规范**
- 命名不符合项目约定
- 代码重复（DRY原则）
- 过长的方法/类（需拆分）
- 缺少必要注释

**配置问题**
- 环境变量未使用
- 硬编码配置
- 缺少错误处理

#### 🟢 Minor（可选优化）

**风格问题**
- 代码格式不一致
- 未使用的变量/导入
- console.log未清理

**可维护性**
- 魔法数字未提取为常量
- 缺少单元测试
- 文档不完整

### 3. 输出审查报告

**报告格式**：

```markdown
## Code Review 报告

### 变更概览
- 新增文件：X 个
- 修改文件：Y 个
- 总变更：+XXX 行 / -YYY 行

### 🔴 Critical（需要修复）

1. **[问题标题]** — `文件名:行号`
   问题描述
   建议：具体修复方案

### 🟡 Important（建议修复）

2. **[问题标题]** — `文件名:行号`
   问题描述
   建议：具体修复方案

### 🟢 Minor（可选优化）

3. **[问题标题]** — `文件名:行号`
   问题描述
   建议：具体修复方案

### 总结
整体评价 + 关键建议
```

## 审查重点（按语言）

### JavaScript/TypeScript
- 异步操作错误处理（try-catch）
- Promise未捕获异常
- 内存泄漏（事件监听器未移除）
- 类型安全（TypeScript）

### Python
- 异常处理过于宽泛（bare except）
- 可变默认参数
- 资源未关闭（文件、数据库连接）
- 类型提示缺失

### Java
- 空指针异常风险
- 资源未关闭（try-with-resources）
- 并发安全问题
- 异常处理不当

### Vue/React
- 组件状态管理
- 生命周期钩子使用
- Props验证
- 内存泄漏（定时器、事件监听）

## 审查清单快速参考

使用时快速过一遍：

- [ ] 安全：输入验证、权限校验、敏感信息
- [ ] 性能：N+1查询、内存泄漏、不必要计算
- [ ] 健壮性：空值防御、错误处理、边界条件
- [ ] 规范：命名、格式、注释、测试
- [ ] 可维护性：代码重复、复杂度、文档

## 保存审查报告

将审查报告保存到：
```
docs/review/YYYY-MM-DD-<topic>-review.md
```

## 注意事项

1. **客观专业**：指出问题，不批评人
2. **具体可行**：给出文件名、行号、修复建议
3. **优先级明确**：Critical必须修，Minor可选
4. **保持平衡**：既要严格也要认可好的实现
5. **参考项目规范**：检查是否符合AGENTS.md中的约定
